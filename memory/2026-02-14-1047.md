## 為什麼「完全無後端」會卡住？

### 1) TDX 授權需要 `client_secret`（Client Credentials）
TDX 的 Access Token 是用 OIDC `client_credentials` 拿到的（SampleCode 就是這個流程）：
- 你必須用 `TDX_CLIENT_ID` + `TDX_CLIENT_SECRET` 去打 token endpoint 才能拿到 `access_token`
- **如果只有前端**，就代表你必須把 `TDX_CLIENT_SECRET` 放在 HTML/JS 裡（或讓使用者在頁面輸入）
- 這會導致：
  - 任何人打開 DevTools 就能看到 secret
  - 你的 secret 會被盜用、額度被刷爆（你已經遇到 429 了）

結論：**純前端可以做，但等同於把 API 密碼公開**（不建議）。

### 2) `.env` 無法在純 HTML 中安全使用
`.env` 是 Node/後端或 build 工具的概念：
- 你「單一 HTML」直接跑，瀏覽器**不會讀** `.env`
- 你想用的話只能：
  - 手動把 key 填在頁面上
  - 或用 build step 把它注入（那又回到工具鏈/非單檔）

### 3) CORS / file:// 來源問題（可能踩雷）
即使你願意把 secret 放前端，瀏覽器還可能因為：
- token endpoint / api endpoint 的 **CORS policy**
- 你用 `file://` 直接開 HTML 時，origin 是 `null`
而被擋請求。通常要「單檔也能跑」會建議至少用一個 static server（但你說不要後端）。
