# Runbook: OpenClaw -> Codex Memory Snapshot Sync

## Goal

Let Codex read memory generated by OpenClaw when both run on different machines without direct connectivity.

## Roles

- OpenClaw machine: source-of-truth LanceDB
- Codex machine: read-only replica

## 1) Export on OpenClaw machine

```powershell
pwsh ./skills/codex-openclaw-shared-lancedb/scripts/export-memory-snapshot.ps1 `
  -SourceDbPath "C:/path/to/openclaw/lancedb"
```

Output appears under:
`skills/codex-openclaw-shared-lancedb/assets/memory-snapshot/<snapshotVersion>/`

## 2) Transfer snapshot to Codex machine

Transfer the full `<snapshotVersion>` folder securely.

## 3) Import on Codex machine

```powershell
pwsh ./skills/codex-openclaw-shared-lancedb/scripts/import-memory-snapshot.ps1 `
  -SnapshotDir "C:/path/to/<snapshotVersion>" `
  -TargetDbPath "C:/path/to/codex/local-lancedb"
```

## 4) Validate

- Check import summary output
- Run one recall smoke test with a known anchor memory key

## 5) Operational policy

- Sync before each coding session (or at least daily)
- Keep last 3 snapshot folders
- Never commit raw LanceDB DB content into git

## 6) Troubleshooting

- **Checksum mismatch**: re-transfer snapshot (incomplete file copy)
- **Permission denied**: run with proper filesystem permissions
- **Empty recall**: verify correct TargetDbPath and memory scope
